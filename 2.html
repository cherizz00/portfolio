4. What you‚Äôll learn in the Course?

System Design Basics

Approach to system-design problems: requirements, APIs, data models, components, interfaces, plus CAP, Consistency, Availability, Latency.

Distributed Cache & Database Sharding

Caching strategies, eviction policies, consistent hashing, Zookeeper, pooling & streaming, Pub/Sub patterns, MapReduce fundamentals.

Distributed Systems Core

Multiprocessor architecture, load balancers, distributed caches, CDNs, push/pull mechanisms, CAP theorem, fault tolerance.

Microservice Architecture

Monolithic vs. microservices, fault tolerance, service discovery, distributed tracing, monitoring, and observability tools.

Cloud Deployment Strategies

AWS EC2 setup, Docker containerization, cloud-native best practices, Kubernetes Pods, CI/CD pipelines, container orchestration.

Microservice HLD & LLD with Kafka

Design microservices using Kafka, message queues, API Gateways, service meshes, and distributed event-driven patterns.

Design Patterns Deep Dive

SOLID principles, Singleton, Factory & Abstract Factory, Builder, Observer, Strategy, and when/how to apply them in large systems.

Scalable-System Case Studies

Real-world designs: TinyURL, Uber backend, Facebook News Feed, Google Maps, Swiggy‚Äôs geo-based architecture, Gmail & WhatsApp scale.

SQL vs. NoSQL Databases

Relational vs. key/value vs. document vs. graph: Cassandra, MongoDB, Redis ‚Äî choosing the right storage for scale & consistency.

High-Level & Low-Level Design

Component diagrams, microservice interactions, data flow, detailed class/method design, and breaking monoliths into services.

Kubernetes & Docker Orchestration

Container scaling (vertical & horizontal), Helm charts, AWS EKS/GKE, WebSockets, long-polling, and rolling updates in production.

Real System-Design Interview Prep

Hands-on mock questions, trade-off analysis, performance bottlenecks, capacity planning, and whiteboard-style problem solving.
